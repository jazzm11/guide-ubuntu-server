Guide: Oppsett av Ubuntu Webserver (Apache / NGINX)

Denne guiden er laget for å gi en ny lærling en praktisk og forståelig steg-for-steg gjennomgang av hvordan man setter opp en Ubuntu-basert webserver. Guiden dekker alt fra å opprette en virtuell maskin (VM), installere Ubuntu Server, til å konfigurere enten Apache eller NGINX som webserver. Til slutt viser guiden hvordan man kan hoste innhold på serveren, inkludert denne guiden selv. Hensikten er å gi lærlingen praktisk erfaring med serveradministrasjon, forståelse for webserver-konfigurasjon, og trygghet i å publisere innhold på en egen server. Dette gir et solid grunnlag for videre arbeid med nettsider, systemadministrasjon og IT-drift.

1. Opprette VM 2. Installere Ubuntu Server 3. Bruk SSH 4. Instalere NGINX 5. Lag og Hoste nettsiden

1. Opprett VM

• Gå til Proxmox: pve-backend.ikt-fag.no Proxmox er et program som lar deg kjøre flere virtuelle maskiner på én server. Du kan bruke det til å lage og styre VM-er, containere og servere via et enkelt web-grensesnitt.

• Trykk på Create VM øverst til høyre.

• Skriv navnet til VM-en og legg til din pool. En pool er en samling ressurser som deles og brukes sammen som for eksempel Server/Virtualisering og Database/Tilkoblinger
• Velg Ubuntu 24.04 Operativsystem: Ubuntu er et gratis og åpent Linux‑basert OS som brukes på servere, PC­er og virtuelle maskiner.
• Bytt Machine til q35 → trykk Next. Machine → Q35: Gir støtte for moderne maskinvare, PCI‑Express og nyere funksjoner, som gjør at VM‑en kan kjøre ny programvare bedre.

• Ikke endre noe på Disk.

• På CPU → endre kjerner til 4 og Type til host. Type → Host: VM‑en arver maskinvaren fra vertsmaskinen, noe som gir bedre ytelse og kompatibilitet.
• Sett RAM til 4096 MiB. 4 MiB RAM brukes for å gi VM-en nok minne til å starte og kjøre lett, samtidig som man sparar ressurser på vertsmaskinen.

• Hopp over Network.

• Trykk Finish.

2. Installer Ubuntu Server

• Start VM → velg Try or Install Ubuntu Server Ubuntu Server er laget for å kjøre tjenester og applikasjoner på en server, uten grafisk brukergrensesnitt. Bruksområder: Webservere (Apache, NGINX) Database-servere (MySQL, PostgreSQL) Fil- og lagringsservere Virtuelle maskiner og cloud-tjenester

• Velg språk: English

• Keyboard layout → Norwegian

• Velg Ubuntu Server for Installasjon

• Gå til Network Configuration → IPv4 → Method: Manual IPv4: Det er det mest vanlige og kompatible protokollformatet for nettverk. Manual: Slik kan du sette en fast IP-adresse, som gjør det enklere å nå serveren og konfigurere tjenester.
Sett IP: Du setter IPv4 til manual for å gi VM-en en fast IP-adresse i stedet for at den får en tilfeldig fra DHCP. 
Subnet: 10.12.[pool].0/24
Address: 10.12.[pool].180
Gateway: 10.12.[pool].1
DNS: 10.12.[pool].255, 1.1.1.1
Search domain: ikt-fag.no

• Hopp over Proxy og Mirror.

• Velg Standard storage.

• Lag bruker og hostname. Bruker: Gir deg et personlig og sikkert konto å logge inn med, i stedet for å bruke root hele tiden. Hostname: Gir VM-en et unik navn på nettverket, så du kan identifisere den enkelt blant andre maskiner.

• Skip Ubuntu Pro.

• Installer SSH. Vi laster ned/installere SSH for å kunne logge inn og styre serveren eksternt på en sikker måte, uten å måtte bruke fysisk skjerm eller tastatur.

• Reboot → trykk Enter. Vent til Rebooten er ferdig og trykk Enter.

3. Bruk SSH

Åpne Terminal på PC-en din og skriv:

ssh brukernavn@10.12.[pool].180

Svar yes og skriv passordet ditt.

4. Installer Webserver (NGINX-VM)

Oppdater systemet og installer NGINX: NGINX er en populær og effektiv webserver som brukes til å levere nettsider og applikasjoner. 
sudo apt update
sudo apt install nginx -y

Start NGINX:

sudo systemctl start nginx

Aktiver auto-boot:

sudo systemctl enable nginx

Sjekk status:

sudo systemctl status nginx

Test i nettleseren:

http://10.12.[pool].180

5. Lag og host nettsiden

Gå til webroot:

cd /var/www/

Gi deg selv eierskap:

sudo chown -R $USER:$USER /var/www

Klon git-repo:

git clone https://gitlab.ikt-fag.no/mcp/guide-ubuntu-server.git

Rediger NGINX config:

sudo nano /etc/nginx/sites-available/default

Endre:

root /var/www/html;

Til:

root /var/www/guide-ubuntu-server;

Restart NGINX:

sudo systemctl restart nginx

Test nettsiden:

http://10.12.[pool].180

Klone Repo fra Gitlab/GitHub

git clone [lenke til repo eks: https://gitlab.ikt-fag.no/mcp/guide-ubuntu-server.git]
